OOP Part 1

What is OOP?

Object-Oriented Programming (OOP) is the representation of a program as objects. The Java language is completely based upon everything being represented as classes and objects. Objects are created by classes, which serve as a template for objects made of that class type. Objects are created by calling the class's constructor, which makes an instance of an object.

If you have a class 'Animal', you may make an Animal object by calling its constructor with the same name, with parentheses to specify the parameters: 

 Animal doggy = new Animal();

In this example, the class is Animal, the object is doggy, and doggy is set to an Animal created by the constructor Animal() with no parameters.

A class consists of attributes and behaviors, known more formally as variables and methods respectively.

Variables can either be instance or local, which describes the scope of the variables. Instance means they are defined at the class level, anything inside the class may use the variable since it has been defined outside of any blocks. Local means the variable is defined in some sort of block (of curly braces '{}'), which indicates the variable only exists within the occurrence of the block. This happens most commonly inside methods, but may also happen inside loops or conditional statements.

In an OOP approach to program design, Methods are often classified as accessors or mutators. Accessors (AKA get methods) retrieve the value of a variable. Mutators (AKA set methods) change the value of a variable. Get methods always start with 'get' and are followed by the variable name, and set methods start with 'set'. The reason why these exist will be explained in OOP Part 2.

Inheritance

Of the four OOP principles, inheritance is one of the more complex principles. Inheritance is when subclasses inherit attributes and behaviours from super classes. All subclasses will be of the type superclass and have all of their variables and methods. Inheritance is done in Java by using the extends keyword when defining a class. When doing this, the class will inherit all variables and methods from the class that is being extended. Here is an example of a class being extended: 

 public class Animal {
     private String name;
     public void sayName() {
         System.out.println(name);
     }
 }
 public class Dog extends Animal {
     public void bark() {
         System.out.println("Ruff!");
     }
 }

The following actions can be made: 


 Dog rex = new Dog();
 rex.sayName();
 rex.bark();
Keyword Definition
Variable An attribute that stores a value
Method A behaviour an object can perform
Superclass A parent class that has at least one subclass that inherits from it
Subclass A class that inherits from a superclass
